# 什么是提示词工程（Prompt Engineering）？

**Prompt Engineering**（提示词工程）= 设计有效的输入文本，让 AI 给出更好的回答。

:::tip 一句话理解
你问得好不好，决定了 AI 回答得好不好。
:::

## 基础概念

### 什么是 Prompt（提示词）？

**Prompt** = 你给 AI 的输入文本。

```
你 → "解释什么是机器学习" → AI → "机器学习是一种..."
          ↑ 这就是 Prompt
```

### 为什么要学习 Prompt Engineering？

因为同样的意思，不同说法，AI 的回答质量天差地别。

**普通提问：**
```
写一段关于猫的文字。
```

**经过 Prompt Engineering：**
```
请用简单易懂的语言（面向 10 岁小朋友）写一段关于猫的科普文字，
包含猫为什么喜欢抓东西。
```

## 核心技巧

### 1. 角色扮演（Role Prompting）

让 AI 扮演某个角色。

```
你是一位经验丰富的 Python 导师。
请用通俗的语言解释什么是列表推导式（List Comprehension）。
```

### 2. Few-Shot 提示

给几个例子，让 AI 模仿。

```
把下面的中文翻译成英文：

中文：你好
英文：Hello

中文：谢谢
英文：Thank you

中文：今天天气真好
英文：
```

### 3. Chain-of-Thought（思维链）

让 AI 分步思考。

```
请逐步思考：
小明有 3 个苹果，妈妈给他 5 个，他吃掉 2 个，还剩几个？
```

### 4. 结构化输出

要求 AI 按格式输出。

```
请总结这篇文章，按以下格式输出：
- 核心观点：
- 关键论据：
- 结论：
```

## 高级技巧

### RAG（检索增强生成）

让 AI 基于你的资料回答，而不是乱编。见 [RAG 详解](./what-is-rag)。

### 系统提示（System Prompt）

预设 AI 的行为模式。

```
你是一个简洁的技术文档翻译者。
翻译时保持专业术语原样不翻译，只翻译解释部分。
```

## Prompt 模板示例

### 学习助手

```
你是一个耐心的 AI 老师。学生问：{学生的问题}
请用 {简单/中等/深入} 的难度解释，并举一个生活化的例子。
```

### 代码解释

```
请解释这段代码：
{代码}

要求：
1. 每一行都解释
2. 说明整体逻辑
3. 指出潜在问题
```

### 写作助手

```
帮我写一篇关于 {主题} 的文章。
风格：{专业/轻松/学术}
字数：{500/1000} 字
目标读者：{学生/职场人士}
```
